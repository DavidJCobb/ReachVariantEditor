Team Configuration

The team configuration page allows you to edit data related to each of a game variant's eight teams.

The fields available to edit are as follows:

Name
Allows you to override a team's name with custom text. The team name cannot be longer than 31 characters.
Enabled
Controls whether this team is active, and whether players are allowed to be on it.
Set Primary Color
If enabled, allows you to override a team's primary color. The primary color is used for players' armor color and for the color of any displayed objectives. Note that dark colors are likely to appear differently in-game due to the shaders used.
Set Secondary Color
If enabled, allows you to override a team's secondary color.
Set Tertiary Color
If enabled, allows you to override a team's tertiary color.
Player Species
If the general team settings are set to have players use team-specific species, then this option controls whether this team's players spawn as Spartans or Elites.
Fireteam Count
Determines the number of "fireteams" on this team.
Initial Designator

Determines how this team definition is mapped to the list of teams in-game. Valid designators range from 1 to 8. The values chosen for all enabled teams should not leave any gaps.

After each round in a match, teams will swap places; for example, in offense/defense game variants with two teams, each team will take turns playing the "defense" role. The game engine takes care of this behavior by remapping teams internally: the gametype script always treats "Red Team" (or team[0] internally) as Defense, and the game engine just lies about which team is "Red Team."

This option, then, determines a team's initial identity. Which team is "Red Team," or team[0], or "1," in the first round?
